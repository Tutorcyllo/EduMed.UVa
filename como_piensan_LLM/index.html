Cómo procesa tu pregunta un LLM

Cada vez que pulsas Enviar, ocurren seis pasos en milisegundos. Explóralos uno a uno.

1
El Prompt — la pregunta médica
Punto de entrada del sistema

Todo empieza con texto plano. El modelo no "entiende" el lenguaje como un humano, pero está entrenado para predecir estadísticamente qué texto debería seguir al tuyo.

Prompt de ejemplo
"Paciente de 58 años con dolor torácico opresivo de 2 horas y diaforesis. ¿Cuál es el diagnóstico más probable?"
✓ Un buen prompt incluye
  • Datos demográficos del paciente
  • Síntomas con tiempo de evolución
  • Pregunta clara y específica
  • Contexto clínico relevante
⚠ Un prompt vago genera
  • Respuestas genéricas y poco útiles
  • Mayor riesgo de alucinaciones
  • Necesidad de más aclaraciones
  • Sesgos no deseados
2
Tokenización — de palabras a números
El modelo no lee letras; lee identificadores numéricos

El primer paso es romper el texto en tokens: palabras completas, sílabas o caracteres. Cada token recibe un número único. Prueba con texto médico:

Los tokens aparecerán aquí…

¿Por qué importa? El español y los términos médicos suelen necesitar más tokens por palabra que el inglés. Esto afecta al coste, la velocidad y los límites de contexto del modelo.

3
Vectores — el mapa espacial del significado
Cada concepto es un punto en un espacio de cientos de dimensiones

Cada token se transforma en un vector: una lista de números que representa su significado. Los términos usados en contextos similares aparecen cerca en este espacio matemático. El mapa muestra esto en 2D (en realidad son 768–4096 dimensiones).

Pasa el cursor sobre los puntos para explorar los clusters semánticos.

4
Mecanismo de Atención — el contexto importa
La misma palabra puede significar cosas distintas según la frase

Conclusión clave El modelo no tiene una única representación de "alta" memorizada. La calcula dinámicamente en cada frase, pesando el contexto completo. Por eso entiende que alta significa discharge o high según el entorno.

6
Generación de Respuesta — de probabilidades a texto
El modelo elige el siguiente token más probable, una y otra vez

El modelo no "sabe" la respuesta. En cada paso elige el token más probable dado todo el contexto previo, lo añade y repite. Es generación autoregresiva.
